# 测试清单：品牌换新 + 登录页 + 移动端适配 + 员工权限

> 日期：2026-07-19。针对本轮改动：智选万物品牌、登录页 2:1、移动端外壳（汉堡+抽屉）、员工管理、角色菜单隐藏。

## 一、开始前要准备什么

### 1. 设备与网络
- **一台电脑**（跑着 docker，就是现在这台）+ **一部手机**。
- 手机和电脑要连**同一个 Wi‑Fi / 局域网**。
- 访问地址：
  - 电脑：`http://localhost:18080`
  - **手机：`http://192.168.1.181:18080`**（不能用 localhost，那指向手机自己）
  - 若手机打不开：多半是电脑防火墙拦了 18080 端口，或不在同一网段。可先在电脑上换局域网 IP 试 `http://192.168.1.181:18080` 能否打开来排除。
- 测试时**强制刷新清缓存**：电脑 Cmd+Shift+R；手机可用无痕/隐私窗口。

### 2. 账号（大部分不用提前准备，登录后现建）
| 账号 | 来源 | 用途 |
|---|---|---|
| `admin` / `admin123` | **系统已内置** | 管理员，测全部功能 |
| operator 测试号（如 `op1`） | 登录 admin 后在「员工管理」新建 | 验证只能看业务页 |
| reviewer 测试号（如 `rv1`） | 同上 | 验证角色菜单 |
| publisher 测试号（如 `pb1`） | 同上 | 验证角色菜单 |

> 这些员工号**不需要提前在外部准备**，用 admin 登录后在页面里点几下就能建。建议用简单密码如 `test1234` 方便测试。

### 3. 外部服务凭证（可选，仅测"真实集成"才需要）
纯 UI / 权限 / 移动端测试**不需要**这些。只有你想验证真实爬虫/上架/AI 时才准备：
- Ozon Seller：`Client-Id` + `Api-Key`
- LLM：`api_key` + `base_url`（如 OpenAI 兼容端点）
- 生图服务凭证、1688 cookie 等
- 填写入口：登录 admin →各「配置」页。

### B. 移动端主界面（本轮新增重点）
- [ ] 手机登录后，顶部是深色条：**左上角 ☰**、logo、智选万物。

- [ ] 抽屉底部「退出登录」→ 回到登录页。

### C. 角色权限与菜单隐藏（安全重点）
- [ ] 用 **admin** 登录：菜单能看到全部 14 项（含 店铺管理/定价/各配置/**员工管理**）。

### D. 员工管理（admin）
- [ ] 「员工管理」页：表格显示 用户名 / 角色（中文 Tag：管理员·操作员·审核员·发布员）/ 状态 / 创建时间 / 操作。
- [ ] **新增员工**：填用户名+密码+角色 → 列表出现新行。
- [ ] 用新建的号能**登录成功**。
- [ ] **改角色**：把 op1 改成 reviewer → 再用 op1 登录，菜单/权限随之变（角色实时生效）。
- [ ] **重置密码**：给 op1 设新密码 → 旧密码登录失败、新密码成功。
- [ ] **停用**：停用 op1 → op1 无法登录。
- [ ] **删除**：删掉一个测试号 → 列表消失。
- [ ] **防锁死（重要）**，以下操作应被后端拒绝并弹中文提示：
  - [ ] 停用 / 降级 / 删除**最后一个管理员** → 拒绝「至少保留一个可用管理员」。
  - [ ] 停用 / 删除**当前登录的自己** → 拒绝。
