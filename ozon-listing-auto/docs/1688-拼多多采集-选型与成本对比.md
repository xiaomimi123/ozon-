# 1688 / 拼多多 采集 —— 技术选型与成本对比报告

> 日期：2026-07-20。用户已定方向：**指纹浏览器硬爬为主**；反爬**尽量不触发 + 需要时接打码 API**；本报告为选型/成本依据，供拍板后进入设计。
> 价格均为调研时区间，**下单以各官网实时为准**。

## 0. 结论摘要

> **注**：§0–§9 是初版（指纹浏览器 + 住宅代理 + 打码）。用户后续反馈"API 成本不接受、可自购 socks5、想用 Chrome 插件"，**最终推荐见 §10（省钱版：自建采集插件 + AdsPower + 自购 socks5）**，把成本从 §0 的 $70–300 压到 ~$15–30 + 你的 socks5。以下 §0–§9 作为背景与对照保留。

- 主路线：**指纹浏览器**。省钱版技术栈（§10 最终）：**自建 content-script 采集扩展（拦截页面接口响应，绕签名逆向）+ AdsPower 无头(装扩展) + 自购 socks5 + 账号池换号 + 偶发 CapSolver 打码**，复用我们已建的**货源账号池**（Cookie/限速/日限/冷却换号）。
- 初版成本（住宅代理版）：大头是住宅代理带宽，月约 **$70–300**；**省钱版（socks5 自购）压到 ~$15–30 + socks5 费**（见 §10.3）。
- **诚实提醒**：硬爬是持续军备竞赛（尤其拼多多 `anti_content` 常变）+ ToS/法律灰色。**没有免认证的免费 1688 API**（§10.1）；1688 有**官方免费 Chrome 插件**可作零开发兜底，官方 API 需企业/分销资质、免调用费但非 drop-in。

## 1. 各平台数据源现状

| 平台 | 官方 API | 硬爬难度 | 推荐取数方式 |
|---|---|---|---|
| **1688** | ✅ 开放平台有关键词搜索/拍立淘图搜/详情/**跨境寻源代采(免调用费+GMV门槛)**（OAuth2 企业认证） | 中（有账号池+端点可配骨架） | 官方 API 最稳；硬爬作兜底 |
| **拼多多** | ❌ 无好用的开放数据 API | 高（`anti_content` 由 4 段 JS 加密、需逆向且常变；列表需登录态 AccessToken 或高质量 IP） | 指纹浏览器渲染取 API 响应；或 onebound 聚合 API |

## 2. 指纹浏览器选型（硬爬核心）

| | AdsPower | BitBrowser（比特） |
|---|---|---|
| 免费额度 | 2 个环境 | 10 个配置文件 |
| 付费起步 | Base $9/月（10 环境）/ Pro $20/月（100 环境） | $10/月（50 档案）起；$140/月（3000 档案/30 用户） |
| Local API | ✅ 支持 Playwright/Puppeteer/Selenium | ✅ 支持，内置 RPA |
| 服务器/无头 | ✅ **官方支持 Linux/Docker 无头、多服务器部署** | ✅ 支持 |
| 定位 | 功能全、用户基数大、文档好 | 更便宜、RPA 强、性价比高 |

**推荐 AdsPower**（服务器无头 + Local API 成熟、文档好）。多账号量大再看 BitBrowser 性价比。我们只需少量账号（账号池换号），Base/Pro 档足够。

## 3. 自动化框架

- **Playwright（Python）**：2026 首选，`playwright-stealth` 维护活跃、context-manager API。经 AdsPower Local API 拿到调试端口后用 Playwright 接管。
- Stealth 只补指纹层，**不解决 IP 信誉/TLS 指纹/行为分析/CDP `Runtime.enable` 泄漏**——所以必须叠**住宅代理 + 拟人节奏**，指纹浏览器本身也帮补真实指纹。

## 4. 住宅代理选型（成本大头）

- 按 GB 计费，量越大越便宜：约 **$6/GB（1–9GB）→ $3–4/GB（30–199GB）→ ~$2/GB（1000GB+）**（2Captcha residential 为参考档）。同类：Bright Data、IPRoyal、Oxylabs 等。
- **浏览器渲染整页很费流量**（每商品页含图/JS 常 2–5MB+），这是硬爬成本主因。可优化：拦截图片/字体/媒体请求只取数据接口响应、复用会话、只在必要页开浏览器。

## 5. 打码 / 过验证码 API（仅触发时用）

| 服务 | 价格（参考） | 说明 |
|---|---|---|
| **CapSolver** | reCAPTCHA 低至 **$0.65/1000**；滑块类似低价 | AI 自动，速度快，支持滑块/reCAPTCHA/Cloudflare 等 |
| **2Captcha** | 普通图形 $0.5–1/1000；reCAPTCHA v2 ~$2.99/1000 | 人工+AI，语言 SDK 全 |
| **YesCaptcha / NextCaptcha** | 类似区间 | 备选 |
| 自建（开源） | 仅服务器成本 | [chenwei-zhao/captcha-recognizer](https://github.com/chenwei-zhao/captcha-recognizer) 滑块缺口检测，可自部署，省调用费但要自维护 |

**策略**：先靠"不触发"（住宅代理 + 限速 + 换号）压低验证码出现率；真出滑块再调 CapSolver。打码成本通常很低（每月几美元级），**前提是触发率低**。

## 6. 成本对照：硬爬 vs API（中小规模，采 ~1万商品/月）

| 项 | 指纹浏览器硬爬 | API 路线（1688官方 / onebound 聚合） |
|---|---|---|
| 指纹浏览器 | AdsPower $9–20/月 | — |
| 住宅代理 | **$60–250/月**（带宽主导，可优化） | 极少 / 无 |
| 打码 API | 几美元/月（触发率低时） | 无 |
| 数据 API 费 | — | 1688官方跨境寻源**免调用费**(需 GMV)；onebound 按量（约数百–上千元/月，量定，**以官网为准**） |
| 维护成本 | **高**（平台改版/anti_content 变即失效，需持续跟） | 低（provider 维护接口） |
| 合规风险 | ToS/法律**灰色** | 官方/授权，**低** |
| **月成本量级** | **≈ $70–300**（代理主导） | 视调用量，常与硬爬相当或更低，但更省人力 |

> 要点：硬爬省的是"数据 API 费"，但**用代理带宽 + 持续维护人力换**。规模上来后，1688 官方 API（免调用费）几乎总是更划算、更稳——**强烈建议 1688 至少并存一条 API 源**。

## 7. 与现有架构衔接（不重造）

我们已有 `SourceProvider` 抽象 + `factory`(ali1688/pinduoduo) + **货源账号池**（Cookie/限速/日限/冷却换号/风控置冷却）+ 端点可配。硬爬方案落地为一个 **provider 实现**：

- `FingerprintBrowserProvider`（新）：调 AdsPower Local API 启动/接管某账号的无头环境 → Playwright 打开搜索/图搜/详情 → 拦截并取数据接口响应 → 解析成统一 offer 结构。
- 用账号池 `acquire()` 取 Cookie 账号、命中风控/验证码时 `report_risk()` 置冷却换号（**已实现**）。
- 验证码：provider 内遇滑块 → 调 CapSolver → 回填。
- 配置：在「货源配置」加"采集方式=API / 指纹浏览器"开关、AdsPower/CapSolver/代理 的凭证字段（沿用 Fernet 加密 + 脱敏 + 高级折叠）。
- 拼多多同理再加一个 provider；`anti_content` 走"浏览器真实渲染后取接口响应"绕过逆向（渲染态天然带合法 anti_content），比纯逆向更耐变。

## 8. 风险与合规（务必知悉）

- **ToS**：1688/拼多多用户协议一般禁止自动化抓取；硬爬属灰色，平台可封号/封 IP。→ 用**独立小号 + 换号 + 限速**降低单号风险，**别用主力账号**。
- **法律**：抓取公开商品信息风险相对可控，但**规模化、绕过技术措施**在部分司法辖区有争议；避免抓取个人信息/破坏服务。
- **稳定性**：拼多多防护变化快，预留"接口失效告警 + 快速改配"能力（我们端点/参数可配已具备）。
- **降级**：任何一层（代理/打码/指纹环境）故障，provider 应优雅失败并让账号池换号，不阻塞其它任务（对齐现有单条隔离）。

## 9. 建议落地步骤

1. **PoC（小验证）**：AdsPower 无头 + Playwright + 1 个 1688 小号 + 1 家住宅代理，跑通"关键词搜索→取接口响应→解析 offer"，遇滑块接 CapSolver 验证一次。评估触发率与单商品流量成本。
2. **provider 化**：把 PoC 封成 `FingerprintBrowserProvider`，接账号池 + 配置页凭证；先 1688。
3. **拼多多 provider**：同法加 PDD（渲染取 anti_content）。
4. **（强烈建议）并存 1688 官方 API 源**作为稳定兜底，配置页可切换 API/浏览器。
5. 监控：采集成功率、验证码触发率、代理用量、账号冷却率上看板。

## 10. 补充（2026-07-20 追加）：Chrome 插件路线 + socks5 + 免费 1688 源核实

> 用户反馈：API 成本客户不愿承担；Chrome 有很多采集插件可借鉴；可自购 **socks5** 代理；希望找免费 1688 API 加入。据此更新如下。

### 10.1 「免费 1688 API」核实结论（诚实）
- **没有"免费、免认证、拿来即用"的 1688 商品 API。** 官方开放平台（[open.1688.com](https://open.1688.com/)）**免调用费**，但商品/搜索类接口多需**开发者应用 + 企业认证/分销资质**，且免费档限速约 200 次/分（[参考](https://developer.aliyun.com/article/1688333)）。不是 drop-in。
- **真正"免费"的是官方 Chrome 插件**（见下），以及少量社区免费 key（不稳、易失效，不建议依赖）。
- 结论：**1688 走"官方免费插件 / 自建插件采集"比接 API 更现实且零 API 费**；官方 API 仅在你日后拿到分销资质时作稳定兜底。

### 10.2 Chrome 插件路线（**新推荐主力**，最省且最耐反爬）
**为什么插件更聪明**：content script 跑在你**真实登录的页面里**，天然带真实 session/指纹、**页面自己生成的 `anti_content`/签名**——所以**基本绕开签名逆向和大部分机器人检测**（它就是个真人浏览器）。

- **可直接用的官方免费插件**：**「1688 采购助手插件版 / 1688 官方采购助手」**（[chajian.1688.com](https://chajian.1688.com/) / [Chrome 应用商店](https://chromewebstore.google.com/detail/1688%E9%87%87%E8%B4%AD%E5%8A%A9%E6%89%8B%E6%8F%92%E4%BB%B6%E7%89%88/kphldkppgfpjadpabfkghmjbhpcmgpdg)）：**免费**，支持截图搜同款(跨 1688/淘宝/亚马逊/京东/**拼多多**)、素材批量下载、**整店商品导出 Excel**、图搜后**一键铺货/分销**。→ **零开发的临时免费方案**：人工用它导出 Excel，再导入我们系统。适合起步/小量。
- **自建采集插件（自动化主力）**：自己写一个 content-script 扩展，在 1688/拼多多页面**拦截页面自身的 XHR/fetch 响应**（搜索/详情接口的 JSON，天然带合法签名），结构化后 POST 回我们后端。比 DOM 解析更稳（拿的是结构化数据，非 HTML）。可借鉴 [Web Scraper](https://webscraper.io/)/[Instant Data Scraper](https://chromewebstore.google.com/detail/instant-data-scraper/ofaokhiedipichpaobibbnahnkdoiiah)/[Data Miner](https://dataminer.io/) 的 content-script + 站点模板思路，以及开源 [Asice/chrome-（1688/淘宝采集插件）](https://github.com/Asice/chrome-)。
- **在服务器跑**：把自建扩展**装进 AdsPower 指纹环境**（AdsPower 支持加载扩展 + 无头 Local API），用账号池的 Cookie 登录、Playwright 驱动导航（搜关键词/开详情），扩展拦截响应回传。**socks5 每个环境单独挂**（AdsPower 代理类型支持 **HTTP/HTTPS/SOCKS5**，[配置文档](https://help.adspower.net/docs/k1a68p)）——正好用你自购的 socks5。

### 10.3 你自购 socks5 的适配
- AdsPower 每个浏览器环境可直接填 socks5（host:port:user:pass），[官方支持](https://help.adspower.net/docs/k1a68p)。**质量比协议更重要**：优先**国内住宅/家宽 socks5**（1688/拼多多是国内站，境外 IP 反而更易风控）。
- socks5 替代了原成本表里的"住宅代理"项——**成本大头你自己搞定**，我们这边基本只剩 AdsPower（$9–20/月）+ 偶发打码（几美元/月）。**整体月成本可压到 ~$15–30 + 你的 socks5 费用**。

### 10.4 修订后的推荐栈（省钱版）
**自建 content-script 采集扩展（拦截页面接口响应）+ AdsPower 无头(装扩展) + 你的 socks5 + 账号池换号 + 偶发 CapSolver 打码**；1688 另可随时叠"官方免费插件人工导出"或"官方 API（拿到资质后）"作兜底。**无数据 API 费**。

### 10.5 落地步骤（据此更新 §9）
1. **PoC**：AdsPower 装一个我们写的最小扩展 + 1 个 1688 小号 + 你的一条 socks5 → 打开 1688 搜索页，扩展拦截搜索接口 JSON、回传后端解析成 offer。验证"零签名逆向、低验证码触发"。
2. 扩展加拼多多支持（同法拦截，绕 `anti_content` 逆向）。
3. provider 化：`FingerprintBrowserProvider` 接账号池 + 配置页凭证（AdsPower 端点/CapSolver key/socks5 由账号池带）。
4. 1688 备兜底：官方免费插件人工导出通道 /（可选）官方 API。
